# 워크트리 문서 동기화

워크트리(worktree)에서 작업한 경우, `.gitignore`에 포함된 `docs/workflow/`와 `docs/ai/` 디렉토리의 문서는 git에 추적되지 않으므로 머지 후 워크트리가 정리되면 소실된다. 이를 방지하기 위해 머지 완료 후, 워크트리 정리 전에 해당 문서를 원래 루트 저장소로 복사한다.

## 실행 조건

현재 작업 디렉토리가 워크트리인 경우에만 수행한다. 워크트리 여부는 `git rev-parse --git-common-dir`의 결과가 `.git`이 아닌 경우(예: `/path/to/main-repo/.git`)로 판단한다.

## 동기화 대상 디렉토리

- `docs/workflow/`
- `docs/ai/`

## 동기화 절차

1. **원래 루트 경로 확인**: `git rev-parse --git-common-dir`에서 `.git`을 제거하여 원래 루트 저장소 경로를 얻는다
2. **대상 파일 수집**: 워크트리의 `docs/workflow/`와 `docs/ai/` 디렉토리에서 이번 작업에서 생성/수정된 파일 목록을 확인한다
3. **충돌 검사**: 원래 루트에 동일 경로의 파일이 이미 존재하는지 확인한다
4. **파일 복사**:
   - **충돌 없음**: `cp -r`로 파일을 원래 루트로 복사한다
   - **충돌 발생** (원래 루트에 동일 파일이 이미 존재하고 내용이 다른 경우): 사용자에게 diff를 보여주고 결정을 요청한다:
     ```
     ⚠️ 문서 충돌이 발생했습니다.

     다음 파일이 원래 루트 저장소에 이미 존재하고 내용이 다릅니다:
     - {파일 경로}

     어떻게 처리할까요?

       A) 워크트리 버전으로 덮어쓰기 (이번 작업 내용 유지)
       B) 기존 버전 유지 (원래 루트의 내용 유지)
       C) VSCode diff로 직접 비교하여 수동 병합

     👉 추천: A — 이번 작업에서 생성/수정된 최신 문서를 반영
     ```
   - 사용자가 **C)를 선택**한 경우: `code --diff {원래루트파일} {워크트리파일}`을 실행하여 VSCode diff 에디터를 열고, 사용자가 수동으로 병합할 수 있도록 한다. 병합 완료 확인 후 다음 단계로 진행한다.
5. **복사 완료 확인**: 복사된 파일 목록을 사용자에게 안내한다:
   ```
   📄 워크트리 → 루트 저장소 문서 동기화 완료:
   - docs/workflow/work/backend/spec/{파일명}
   - docs/workflow/work/backend/plans/{파일명}
   - docs/ai/spec/reviews/{파일명}
   ```

## 워크트리가 아닌 경우

이 단계를 건너뛴다.
